## Введение

Разработка проекта, который принимает от пользователя документ формата .docx и с помощью LLM предоставляет некую простую информацию о содержимом файла (основные тезисы, язык документа, простейшая аналитика данных) с обратной передачей в виде файла .docx или .txt на сайте или через email. Осуществляется работа с помощью backend на FastAPI с использованием множеств библеотек, включающие работу с текстом, обработку ее с помощью LLM. Основную структуру frontend представляет собой React для взаимодействия с пользователем через User Interface и обращение с помощью Axios к backend. В функциональность также входит аутентификация пользователей через JWT-токены. Авторизованные пользователи могут посредством обращения к функциональности сайта делать ограниченное количество запросов по принятию .docx файла, так как реализован race-limiter. Все готовые форматы файлов каждого пользователя лежат в S3 хранилище, а сами данные пользователей в БД с использованием PostgreSQL. Для поднятия БД(ревизии и миграции) реализованы через Alembic, а также используется ORM в виде SQLAlchemy для реализации аналатики данных. Проксирование и скрытие портов фронтенда и бэкенда сделано через веб-сервер Nginx, а также реализована раздельная передача сообщений через брокер сообщений RabbitMQ для разбиения монолита backend на микросервисы. Проект запускается в пару кликов локально с использованием готового docker-compose. Данный проект обработан линтерами, форматерами и настроен CI, а также написаны тесты backend на pytest.

## Цель проекта

Цель проекта познакомиться на практике с простейшей реализацией функциональности LLM и разделение проекта с монолитной структуры на микросервисы для искусственого взаимодействия с RabbitMQ. Важным аспектом является умение поработать с хранилищем данных S3 для .docx и .txt файлов для каждого пользователя. Также для меня важно усвоить лучше навыки frontend-разработки, чтобы разобраться еще глубже в процессах React. Интересно поработать с файлами для отправки через backend и frontend, а также для взаимодействия с библеотекой почты.
